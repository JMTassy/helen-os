## 3. LES DEUX ACTIFS COUPLÉS

### CONQUEST — le produit (acquisition)

- Jeu physique (30 000 boîtes) : produit d'appel, media tangible, marge brute ~50%
- Application mobile (Dream + Life + Love + IA) : récurrence, conversion, engagement
- Launch : entonnoir B2C, presse, communauté, partenariats

CONQUEST seul, sans HELEN OS = jeu hybride sans moat. Remplaçable.

### HELEN OS — le moteur (multiplicateur de valeur)

- Historique utilisateur exploitable (mémoire persistante entre sessions)
- Expérience non reproductible (personnalisation par joueur)
- Traçabilité native (gouvernance par reçus, audit trail, conforme par construction)
- Continuité = rétention structurelle

HELEN OS sans CONQUEST = infrastructure invisible au marché.

### Le couplage fait la défense

```
CONQUEST ─── acquisition, récurrence, narrative
    │
    ▼
HELEN OS ─── mémoire, reçus, personnalisation
    │
    ▼
MOAT     ─── non réplicable par un éditeur logiciel pur
             non réplicable par un studio physique pur
```

**Sans HELEN → LTV plafonnée. Avec HELEN → LTV croissante.**

Un concurrent peut copier le jeu. Il ne peut pas copier 8 mois d'historique utilisateur gouverné.
