MCP / COMPUTER USE INTEGRATION
──────────────────────────────

MCP servers are CLAW skills. Each MCP call:
    1. Parsed as ClawAction (skill + payload + rationale)
    2. Gated: known skill → PENDING, unknown → DENY
    3. Converted to kernel proposal (action: claw_external)
    4. Governor validates (write action → PENDING → approval)
    5. On ALLOW: execute MCP call
    6. Receipt includes: MCP server, tool name, params, result hash

Computer Use is a governed visual pipeline:
    screenshot → state observation (C: OBSERVE)
    click target → proposal (C: EXECUTE, target: pixel coordinates)
    governor → PENDING (requires approval for UI mutation)
    execute → perform click (E layer)
    verify → screenshot after (effect_status: MATERIALIZED)
    intent check → did the intended thing happen? (intent_status: UNKNOWN until probed)

    CRITICAL:  pixel changed ≠ task completed
               MATERIALIZED ≠ SUCCESS
               No semantic probe → UNKNOWN
